==================================================================
              LetRecovery - Windows 一键恢复与维护工具
==================================================================

LetRecovery 是一个免费、开源的 Windows 恢复、部署、备份和维护工具。

项目地址：
  https://github.com/NORMAL-EX/LetRecovery

发布下载：
  https://github.com/NORMAL-EX/LetRecovery/releases

------------------------------------------------------------------
主要功能
------------------------------------------------------------------

LetRecovery 面向常见的 Windows 修复和装机场景：

  - 从 WIM、ESD、GHO、ISO 或本地镜像安装 Windows
  - 自动部署并进入内置 WinPE 环境
  - 备份和还原系统镜像
  - 安装过程中导入驱动
  - 修复 Windows 启动项
  - 配置无人值守安装选项
  - 查看硬件和磁盘信息
  - 集中运行常用维护工具

------------------------------------------------------------------
文件说明
------------------------------------------------------------------

  LetRecovery.exe          Windows 端主程序
  config.json              运行配置和 PE 缓存配置
  bin\                     运行组件、语言文件和 PE 文件
  bin\pe\LetRecovery_PE.wim
                           内置 WinPE 镜像，用于离线安装和维护
  README.txt               本说明文件

除非你清楚每个运行组件的用途，否则不要单独移动包内文件。

------------------------------------------------------------------
使用前请注意
------------------------------------------------------------------

系统安装、分区格式化、启动修复、镜像还原等操作可能覆盖数据。
执行这些操作前，请先确认：

  - 重要文件已经备份
  - 目标磁盘和目标分区选择正确
  - 电脑连接稳定电源
  - 安装过程中不要拔出硬盘、U 盘或移动存储设备
  - 如果格式化失败或长时间停在 0%，请先检查硬盘健康状态

如果 DiskPart 报告格式化错误，不建议跳过失败继续安装。
请先检查硬盘、数据线、硬盘盒、控制器、分区表和写保护状态。

------------------------------------------------------------------
建议测试流程
------------------------------------------------------------------

发布包更新后，建议至少测试以下流程：

  1. 从解压后的目录直接启动 LetRecovery.exe。
  2. 确认语言文件和配置文件可以正常加载。
  3. 测试系统安装流程能否正常进入 WinPE。
  4. 确认目标分区会在部署镜像前正确格式化。
  5. 在虚拟机中使用一个已知正常的小镜像完成部署或还原。
  6. 确认启动修复完成后，目标系统可以正常启动。
  7. 如果任一步骤失败，请查看 LetRecovery 或 WinPE 日志。

------------------------------------------------------------------
日志和排查
------------------------------------------------------------------

反馈问题时，建议提供以下信息：

  - LetRecovery 版本或发布包日期
  - 当前启动模式是 UEFI 还是 Legacy
  - 磁盘布局、目标分区和镜像类型
  - LetRecovery 或 WinPE 生成的日志文件
  - 屏幕上显示的完整错误信息

常见安装失败原因包括：镜像损坏、硬盘异常、磁盘写保护、
BitLocker 卷未解锁、WinPE 下盘符变化、驱动缺失等。
